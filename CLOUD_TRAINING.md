# CogNetAICL — Cloud Training Guide

Fine-tune CogNet to write AICL code. This guide covers launching a training
run on a GPU cloud instance from scratch.

## What you're training

CogNet (a character-level language model) is fine-tuned on a corpus of
**AICL spec → generated code** pairs. After training, the model learns to
generate executable AICL/Python/Rust code from a specification.

The corpus is generated by the [AICL compiler](https://github.com/AFKmoney/AICL)'s
`corpus_generator.py` — 30 algorithm templates (sorts, searches, math, string
ops, primes, etc.) written in the AX sub-language, each compiled to real code.

## Prerequisites

- A GPU cloud instance (recommended: 1× NVIDIA GPU with ≥16 GB VRAM)
  - **RunPod**: RTX 4090 or A100 pod, PyTorch template
  - **Lambda Labs**: A10, A100, or H100
  - **Vast.ai**: any RTX 3090/4090 instance
  - **GCP/AWS**: T4, V100, A100
- Python 3.10–3.13
- Git

## Quick start (5 commands)

```bash
# 1. Clone CogNet
git clone https://github.com/AFKmoney/CogNet.git
cd CogNet

# 2. Install dependencies (torch + numpy)
pip install -r requirements_aicl.txt

# 3. Launch training (clones AICL, generates corpus, trains)
python cloud_train.py --steps 5000                     # small model (163M)
python cloud_train.py --steps 5000 --model 1b          # full 1B model
```

That's it. The script auto-clones the AICL compiler, generates a fresh corpus,
and launches `train_pipeline.py` with GPU auto-detection.

## Model sizes

| Flag | Params | VRAM needed | Recommended GPU | Notes |
|---|---|---|---|---|
| `--model small` (default) | 163M | 8–12 GB | RTX 3060, T4 | Faster iteration, good for prototyping |
| `--model 1b` | 1.01B | 24–40 GB | A100 40GB, 2× RTX 4090 | Full capacity, best quality |

Both use seq_len=2048 (full AICL spec context window).

## Training options

```bash
# Full 1B model on A100
python cloud_train.py --steps 5000 --model 1b

# Multi-target corpus (spec → Python AND Rust code)
python cloud_train.py --steps 5000 --model 1b --multitarget

# Small model for quick experiments (fits on consumer GPUs)
python cloud_train.py --steps 5000 --model small

# Resume from latest checkpoint (segment-based, lossless)
python cloud_train.py --steps 5000 --resume

# Skip repo setup if you've already run it once
python cloud_train.py --steps 5000 --skip-setup
```

## Checkpoints

Training saves to `./checkpoints/`:
- `cognet_best.pt` — lowest validation loss (use this one)
- `cognet_latest.pt` — most recent (for resuming)
- `optimizer.pt` — optimizer state (for resuming)

## Monitoring

The pipeline prints loss every `--eval_interval` steps. Watch for:
- **Train loss** decreasing steadily
- **Val loss** decreasing then plateauing (good)
- **Val loss rising** = overfitting (stop and use `cognet_best.pt`)

## After training: test the model

```bash
# Generate text from the fine-tuned model
python infer.py --checkpoint checkpoints/cognet_best.pt --prompt "Goal:\nSort an array\n\nBehavior Sort\n    Input: array\n    Output: array\n    Action:\n"
```

The output should resemble valid AICL + Python code if training succeeded.

## Troubleshooting

**CUDA out of memory**: reduce batch size in `train_pipeline.py` (edit the
default or pass via the pipeline's config). seq_len=2048 needs ~8 GB VRAM.

**Corpus too small**: add more algorithm templates in
`AICL/python/tools/corpus_generator.py` (the `TEMPLATES` list), then re-run
`cloud_train.py`. More diversity = better generalization.

**Model produces garbage**: likely undertrained. Increase `--steps` to 10000+.
Character-level models need many steps to learn structured output.
